**************************************************************

CSVインポート、エクスポートモジュール説明書

**************************************************************

-------------------------------------------------------------
■取込み、出力対象データ
-------------------------------------------------------------
[管理者]
・求職者(nUser)
・企業(cUser)
・中途採用求人(mid)
・新卒採用求人(fresh)

[企業]
・中途採用求人(mid)
・新卒採用求人(fresh)

出力ファイル名は、[テーブル名][年4桁][月2桁][日二桁][時2桁][分2桁][秒2桁].csv
となります。

-------------------------------------------------------------
■使い方
-------------------------------------------------------------
1. 雛形用のデータをシステムで登録します。
2. 一覧の「CSVファイル出力」ボタンを押してファイルに出力します。
3. 出力されたファイルを雛形として、行をコピーして必要なデータを作成します。

注意：
・ヘッダ部分をシステムテーブル項目と照合していますので、ヘッダ行は削除しないで下さい。
・データを新規追加する場合は、id の項目を空にしてください。
・id が設定されている場合は、更新データとしてレコードを更新します。

4. ページの最後の方にある、「CSVインポート」の項目で、CSVファイルを選択すると、
　アップロード&インポートを行ないます。
5. 取込みに成功すると、取込み行数、新規数、更新数を表示します。「ページ更新」
　ボタンを押すと、リストを更新します。

　失敗した場合は、次の項目を確認してください。
・CSVファイルの内容に間違いがないか。
・サーバーのアップロード容量制限を超えていないか
・サーバーの file/upload ディレクトリに書き込み権が設定されているか

-------------------------------------------------------------
■応用
-------------------------------------------------------------

中途採用求人と新卒採用求人は、項目がほぼ同じであり、ヘッダの一部を書き換える事で
データを相互に読み込ませる事ができます。

ヘッダの一番最後の項目は、テーブル名となっており、これにより、別のテーブルの
データを誤って読み込ませる事を防いでいます。

新卒採用求人は fresh、中途採用求人は mid となっていますが、これを nocheck と
書き換えるとどちらのテーブルでも読み込ませる事ができるデータとなります。

注意：
・nocheck を指定すると、読み込み対象のテーブルをチェックしなくなりますので、
　求人広告データを求職者や企業に読み込ませるなど、テーブルの構造が違うテーブル
　への取込みを行わないでください。

-------------------------------------------------------------
■制限事項
-------------------------------------------------------------
・システムのテーブル項目と照合しながら取込みを行なう為、大量のデータを取込む場合、
　膨大な時間が掛かります。バックアップは tool.php や phpMyAdmin を使用するように
　してください。
・企業ユーザーの場合、他社の求人情報のCSV出力を行なえますが、取込んで更新はできません。
・CSVファイルは検索した結果を出力しますので、結果が無い時は「CSVファイル出力」
　ボタンを押してもファイルは出力されません。
